# Profile Page Spec

Цей документ описує, якою має бути сторінка профілю користувача в системі.
Ціль: дати користувачу одне зрозуміле місце, де він може керувати своїми персональними даними, аватаром, сповіщеннями, робочими налаштуваннями та особистими параметрами в межах проєкту.

## Main Goal

Сторінка профілю повинна:

- дати змогу швидко оновити персональні дані
- зробити аватар обов’язковою частиною профілю
- показати роль, повноваження і зону відповідальності
- централізувати налаштування повідомлень
- зберігати робочі налаштування, важливі для задач, часу і комунікації
- бути корисною як для звичайного учасника команди, так і для менеджера

## Recommended Entry Point

Додати в хедері або user menu:

- `Профіль`
- `Сповіщення`
- `Мої закладки`
- `Вийти`

Сторінка профілю:

- `/admin-v2/profile`

## Recommended Page Structure

Сторінка має бути побудована як settings hub з лівим меню або верхніми вкладками.

### Sections

1. `Основне`
2. `Аватар`
3. `Про мене`
4. `Роль і повноваження`
5. `Сповіщення`
6. `Робочі налаштування`
7. `Час і продуктивність`
8. `Безпека`
9. `Інтеграції`
10. `Додатково`

## Section 1 — Основне

Це базова інформація про користувача.

### Fields

- `Ім’я` — обов’язково
- `Прізвище` — обов’язково
- `Повне ім’я` — можна генерувати автоматично
- `Email` — тільки для перегляду або редагування через окремий flow
- `Телефон`
- `Посада`
- `Відділ`
- `Команда`

### Requirements

- ім’я і прізвище не можуть бути порожніми
- значення мають зберігатися окремо, а не лише як один `name`
- у всіх місцях системи показувати нормальне відформатоване full name

## Section 2 — Аватар

Аватар повинен бути обов’язковим.

### Requirements

- якщо аватар відсутній:
  - показати warning
  - не давати завершити онбординг профілю
  - показати placeholder та CTA `Завантажити аватар`
- підтримати:
  - upload
  - crop
  - preview
  - delete/replace

### Constraints

- дозволені формати: `jpg`, `jpeg`, `png`, `webp`
- рекомендований мінімум: `400x400`
- зберігати square version для списків, чату, коментарів, задач

### Why this matters

Для вашого продукту аватар важливий у:

- chat
- comments
- task assignees
- people/workload view
- activity feed

Без нього UI втрачає читабельність.

## Section 3 — Про мене

Блок для короткого професійного опису.

### Fields

- `Короткий опис / Bio`
- `Сильні сторони`
- `Зона відповідальності`
- `Спеціалізація`
- `Навички`
- `Мови`

### Suggested use

Це особливо корисно в цьому проєкті для:

- швидкого розуміння, кого призначати на задачу
- побудови `By People`
- перегляду команди проєкту
- внутрішнього пошуку експертизи

## Section 4 — Роль і повноваження

Користувач має бачити не лише свою роль, а й що вона реально означає.

### Show

- `Системна роль`
- `Роль у компанії`
- `Ролі в проєктах`
- `Ключові дозволи`

### Example permission cards

- може створювати задачі
- може редагувати будь-які задачі
- може призначати виконавців
- може вести час
- може бачити фінанси
- може бачити cost reports
- може керувати webhooks/automations

### Important

Цей блок не обов’язково має бути редагований самим користувачем.
Для більшості ролей він може бути read-only.

## Section 5 — Сповіщення

Це один з найважливіших блоків.

### Channels

- `У системі`
- `Email`
- `Push / Browser`
- `Telegram / Slack` якщо буде інтеграція

### Notification groups

- нові призначення на задачі
- зміна статусу задачі
- коментарі в задачах
- згадки `@mention`
- дедлайн сьогодні
- прострочені задачі
- нові повідомлення в чаті
- зміни у проєктах
- системні події
- фінансові погодження / рев’ю

### UX requirements

- окремі тумблери по категоріях
- `Увімкнути все`
- `Вимкнути все`
- `Лише важливе`
- `Робочий режим без шуму`

### Very useful addition

Додати режим:

- `Не турбувати`
- тихі години
- вихідні дні

## Section 6 — Робочі налаштування

Цей блок повинен підлаштовувати систему під спосіб роботи людини.

### Fields

- `Часовий пояс`
- `Мова інтерфейсу`
- `Формат дати`
- `Перший день тижня`
- `Вигляд задач за замовчуванням`
  - список
  - kanban
  - календар
  - по людях
- `Початковий екран після входу`
  - dashboard
  - мої задачі
  - проєкти
  - чат

### Very useful for this project

- `Проєкт за замовчуванням` для швидкого створення задач
- `Мій збережений view за замовчуванням`
- `Показувати TimerPill`
- `Автовідкривати мої активні задачі`

## Section 7 — Час і продуктивність

Оскільки у вас є timer/time logs/reports, цей блок повинен бути окремим.

### Fields

- `Робочі години`
- `Робочі дні`
- `Денна норма годин`
- `Налаштування таймера`
  - автозупинка попереднього
  - нагадувати, якщо таймер працює довго
  - підтвердження перед stop
- `Налаштування ручного логування часу`
- `Показувати час у моїх задачах`

### Additional useful options

- `Нагадати, якщо сьогодні немає жодного time log`
- `Нагадати в кінці дня закрити незавершений таймер`

## Section 8 — Безпека

### Fields / actions

- зміна пароля
- активні сесії
- вийти з усіх пристроїв
- 2FA
- історія входів

### Suggested additions

- список останніх входів з IP/браузером
- alert на новий незнайомий вхід

## Section 9 — Інтеграції

Цей блок буде дуже корисний для вашої системи.

### Connectors

- Google Calendar
- Slack
- Telegram
- Email forwarding / inbox

### Show

- статус підключення
- підключені акаунти
- остання синхронізація
- доступні дозволи

## Section 10 — Додатково

Сюди варто винести те, що покращує персональну роботу, але не входить у базовий профіль.

### Good additions for this project

- `Мої закладки`
  - збережені views
  - важливі проєкти
  - важливі задачі
  - швидкі звіти
- `Моя завантаженість`
  - коротка статистика open/overdue/done
- `Моя команда`
  - якщо користувач manager
- `Моя активність`
  - останні дії
- `Мої шаблони`
  - якщо будуть personal task templates

Якщо цього ще немає, варто додати або нормалізувати:

- `firstName`
- `lastName`
- `bio`
- `jobTitle`
- `phone`
- `avatarUrl`
- `timezone`
- `locale`
- `dateFormat`
- `weekStartsOn`
- `defaultTaskView`
- `defaultLandingPage`
- `notificationPrefsJson`
- `workPrefsJson`
- `productivityPrefsJson`

## Validation Rules

### Required

- `firstName`
- `lastName`
- `avatar`

### Recommended

- `jobTitle`
- `timezone`
- хоча б один активний notification channel

### Limits

- `bio` до `500-1000` символів
- без HTML
- sanitize user input

## UX Recommendations

### Save behavior

- sticky save bar
- autosave only for small toggles
- явний `Зберегти зміни` для form sections

### Profile completeness

Додати індикатор:

- `Профіль заповнено на 80%`

Критерії:

- ім’я
- прізвище
- аватар
- посада
- опис
- timezone
- notifications configured

### Role-aware rendering

- звичайний користувач бачить базові секції
- менеджер додатково бачить workload/team-related settings
- super admin бачить більше системної інформації

## Recommended UI Layout

### Desktop

- ліва колонка: меню секцій
- права колонка: форма
- зверху sticky profile summary

### Mobile

- accordion або segmented tabs
- summary card зверху

## Suggested Profile Summary Card

Показувати зверху:

- аватар
- ім’я + прізвище
- посада
- команда / відділ
- роль
- короткий bio
- статус профілю

Окремо рекомендую додати:

### 1. Availability status

- `У роботі`
- `На зустрічі`
- `У відпустці`
- `Не турбувати`

Це дуже корисно для chat/tasks/team coordination.

### 2. Expertise tags

Наприклад:

- кошториси
- закупівлі
- інженерія
- фінанси
- виконроб
- дизайн

Це потім можна використовувати для підбору виконавців.

### 3. Personal dashboard preferences

- які KPI показувати
- який focus tab відкривати за замовчуванням
- чи показувати completed tasks

### 4. Quick actions block

- `Мої задачі`
- `Мої проєкти`
- `Мої звіти`
- `Мої закладки`

Потрібні endpoints:

- `GET /api/admin/profile`
- `PATCH /api/admin/profile`
- `POST /api/admin/profile/avatar`
- `DELETE /api/admin/profile/avatar`
- `PATCH /api/admin/profile/notifications`
- `PATCH /api/admin/profile/preferences`
- `PATCH /api/admin/profile/security`

## Definition Of Done

Сторінку профілю можна вважати готовою, якщо:

- користувач може редагувати базові дані
- аватар є обов’язковим і реально використовується в системі
- є окремий блок notification settings
- є role/permission visibility
- є робочі preferences для задач/таймера/виду інтерфейсу
- сторінка корисна і як персональний профіль, і як робочий control center

## Recommended Implementation Order

1. Базові поля профілю
2. Аватар
3. Notification settings
4. Робочі налаштування
5. Role/permission summary
6. Time/productivity settings
7. Інтеграції
8. Security
