# AI Consistency Testing Guide

## Мета тестування

Перевірити що система запобігання AI галюцинаціям працює коректно:

1. ✅ **Детермінізм**: Той самий проект → той самий результат
2. ✅ **Real prices**: AI використовує ціни з бази матеріалів
3. ✅ **Specification compliance**: AI витягує марки та методи зі специфікацій
4. ✅ **Validation**: Система ловить помилки та попереджає
5. ✅ **No hallucinations**: AI не вигадує матеріали/ціни

---

## Тест 1: Детермінізм (Consistency)

**Мета**: Перевірити що той самий проект генерує ідентичні кошториси

### Кроки:

1. Завантаж тестові файли (твої 27 PDF планів котеджу)
2. Заповни wizard:
   - Тип: Котедж (таунхаус)
   - Стан: Коробка (shell)
   - Демонтаж: НІ (demolitionRequired: false)
   - Площа: 150 м²
   - Поверхів: 3
   - Матеріал стін: Газоблок
   - 45 розеток, 23 вимикачі, 12 радіаторів
3. Згенеруй кошторис → Збережи результат як `test1_run1.json`
4. **НЕ ЗАКРИВАЙ СТОРІНКУ**, натисни "Згенерувати" ще раз → `test1_run2.json`
5. Повтори 3-й раз → `test1_run3.json`

### Очікуваний результат:

```bash
# Порівняння
diff test1_run1.json test1_run2.json
# Очікується: < 5% різниці в кількості позицій
# Очікується: Ті самі назви матеріалів
# Допускається: ±10% в кількостях (округлення)
```

**Критерій проходження**:
- ✅ Різниця в кількості позицій < 5% (180 vs 185 = OK, 180 vs 220 = FAIL)
- ✅ Топ-20 матеріалів однакові в усіх 3-х прогонах
- ✅ Загальна вартість різниця < 10%

## Тест 2: Заборона демонтажу (Forbidden Items)

**Мета**: Перевірити що AI НЕ додає демонтаж для shell building

### Кроки:

1. Завантаж плани котеджу
2. Wizard:
   - Стан: **Коробка (shell)** ← КРИТИЧНО
   - Демонтаж: **НІ** ← КРИТИЧНО
3. Згенеруй кошторис

### Очікуваний результат:

```javascript
// У консолі браузера (F12 → Console):
console.log('🔍 Estimate Validation:', json.validation);

// Перевір validation.errors:
// ✅ PASS: errors = [] (пусто)
// ❌ FAIL: errors = [{ code: 'FORBIDDEN_DEMOLITION_ITEM', ... }]
```

**Критерій проходження**:
- ✅ Validation: `errors: []` (немає критичних помилок)
- ✅ Жодної секції з назвою "Демонтаж"
- ✅ Жодної позиції з словами: "демонтаж", "зняття", "видалення"

**Якщо FAIL**:
- Перевір що в wizard правильно вказано `demolitionRequired: false`
- Відкрий `Network` → знайди request `/api/admin/estimates/generate`
- Подивись що `wizardData.houseData.demolitionRequired = false` або `wizardData.townhouseData.demolitionRequired = false`
- Якщо там `true` або `undefined` → проблема в frontend wizard

## Тест 3: Ціни з бази матеріалів (Materials Database)

**Мета**: Перевірити що AI використовує реальні ціни з бази, а не вигадує

### Кроки:

1. Завантаж прості плани (не обов'язково 27, можна 2-3)
2. Wizard: будинок, газоблок стіни, електрика повна
3. Згенеруй кошторис
4. Знайди в кошторисі позицію "Газоблок AEROC"

### Очікуваний результат:

```javascript
// В згенерованому кошторисі знайди:
{
  name: "Газоблок AEROC D400 300х200х600",
  unitPrice: 89, // ← Має бути 89 грн (з бази)
  // НЕ 450, НЕ 120, НЕ 200
}

// Розетка:
{
  name: "Розетка Schneider Electric Asfora",
  unitPrice: 185, // ← Має бути 185 грн (з бази)
  // НЕ 50, НЕ 300
}

// Кабель:
{
  name: "Кабель ВВГ-нг 3×2.5 мм²",
  unitPrice: 42, // ← Має бути 42 грн/м (з бази)
  // НЕ 15, НЕ 80
}
```

**Критерій проходження**:
- ✅ Газоблок AEROC: 89 грн/шт (±10 грн допустимо)
- ✅ Розетка Schneider: 185 грн/шт (±20 грн допустимо)
- ✅ Кабель ВВГ-нг 3×2.5: 42 грн/м (±5 грн допустимо)

**Якщо ціни зовсім інші**:
- Відкрий Console → шукай `💰 Materials database context`
- Має бути: `Materials database context: 8.5KB, categories: foundation, walls, ...`
- Якщо немає → materials context НЕ додався до промпту
- Перевір `/api/admin/estimates/generate` route.ts що `generateMaterialsContext()` викликається

**Мета**: Перевірити що AI витягує точні марки зі специфікацій

### Кроки:

1. Створи тестовий файл `test_specification.txt`:

```

1. ЕЛЕКТРИКА

Матеріали:
- Кабель ВВГ-нг ПівденьКабель 3×2.5 мм² - 250 м
- Кабель ВВГ-нг ПівденьКабель 3×1.5 мм² - 180 м
- Розетка Schneider Electric Asfora біла з заземленням - 45 шт
- Вимикач Schneider Electric Asfora одноклавішний - 23 шт
- Підрозетник D68×45 мм для бетону - 45 шт
- Клеми WAGO 2273-243 (3-контактні) - 135 шт
- Дюбель швидкий монтаж 6×40 мм - 180 шт

Послідовність робіт:
```

2. Завантаж:
   - `test_specification.txt` (специфікація)
   - Плани електрики (якщо є)

4. Згенеруй кошторис

## Тест 5: Валідація (Validation System)

**Мета**: Перевірити що валідація ловить помилки

### 5.1: Нереалістичні кількості

**Штучно створимо помилку** (для тесту):

1. Згенеруй кошторис для 150 м²
2. Якщо AI згенерував 500 розеток → validation має попередити

```javascript
// В консолі після генерації:
console.log('⚠️ Validation Warnings (first 3):');
// Очікується:
// - [EXCESSIVE_QUANTITY] "Розетка ...": кількість 500 виглядає завеликою для 150 м² (зазвичай ~0.3-0.5 на м²)
```

**Критерій проходження**:
- ✅ Warning: `EXCESSIVE_QUANTITY` якщо розеток > 120 для 150 м² (0.8/м²)

### 5.2: Розрахункові помилки

**НЕ можна штучно створити** (AI рахує правильно), але перевір:

```javascript
// Візьми будь-яку позицію з кошторису:
const item = {
  quantity: 45,
  unitPrice: 185,
  laborCost: 2250,
  totalCost: 10575
};

// Перевірка:
const expected = 45 * 185 + 2250; // = 10575
console.log(expected === item.totalCost); // Має бути true

// Якщо false → validation має показати ERROR:
// [CALCULATION_ERROR] "Розетка ...": неправильний totalCost. Має бути 10575, а є XXX
```

**Критерій проходження**:
- ✅ Всі позиції: `totalCost = quantity × unitPrice + laborCost` (похибка < 1 грн)
- ✅ Validation: `errors: []` (якщо розрахунки вірні)

3. Порахуй скільки AI додав розеток в кошторис


**Критерій проходження**:
- ✅ Розеток в кошторисі: 45 (±30% = 31-59 шт допустимо)

## Troubleshooting (Якщо тести FAIL)

### Тест 1 FAIL (Непостійність):
- Перевір що temperature = 0.1 (не 0.3)
- Перевір що responseMimeType = "application/json" для Gemini
- Можлива причина: різні файли завантажені між прогонами

### Тест 2 FAIL (Є демонтаж):
- Перевір wizard: `demolitionRequired` має бути `false`
- Перевір `/api/admin/estimates/generate` → wizard context має містити "БЕЗ демонтажу"
- Перевір validation: має бути ERROR з кодом `FORBIDDEN_DEMOLITION_ITEM`

### Тест 3 FAIL (Неправильні ціни):
- Перевір Console → `💰 Materials database context: X KB`
- Якщо немає → materials context НЕ додається
- Перевір `generate/route.ts` → `generateMaterialsContext()` викликається?
- Перевір prompt → має бути секція "💰💰💰 БАЗА МАТЕРІАЛІВ"

### Тест 4 FAIL (Немає марок):
- Перевір назву файлу: `specification.txt`, `spec.pdf`, `технологічні_карти.pdf`
- Перевір Console → `📚 Specification context: X KB`
- Якщо немає → файл не класифікувався як специфікація
- Збільш розмір файлу > 10MB АБО додай "специф" в назву

### Тест 5 FAIL (Валідація не працює):
- Перевір Console → `🔍 Estimate Validation:`
- Якщо немає → валідація не інтегрована
- Перевір `generate/route.ts` → `validateEstimate()` викликається перед return?

1. Збережи console output (F12 → Console → Right click → Save as...)
2. Збережи згенерований кошторис (JSON)
3. Збережи wizard дані (якщо є)
4. Створи GitHub Issue з:
   - Назва тесту що провалився
   - Console output
