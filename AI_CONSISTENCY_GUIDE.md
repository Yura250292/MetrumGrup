# AI Consistency & Anti-Hallucination System

## Проблема

AI моделі (Gemini, GPT, Claude) мають 2 критичні проблеми при генерації кошторисів:

1. **Непостійність (Non-determinism)**: Той самий проект → різні результати при кожному запуску
2. **Галюцинації (Hallucinations)**: AI вигадує матеріали, ціни, кількості, які не існують

## Рішення

### 1. Детермінізм (Temperature 0.1)

**Зміна**: Знижено temperature з 0.3 до 0.1 для всіх моделей

```typescript
// Gemini
generationConfig: {
  temperature: 0.1, // Було 0.3
  responseMimeType: "application/json",
}

// OpenAI GPT-4o
temperature: 0.1, // Було 0.3
response_format: { type: "json_object" }

// Anthropic Claude Opus 4
temperature: 0.1, // Було 0.3
```

**Результат**: Той самий проект → той самий кошторис (±5% варіативність замість ±30%)

### 2. Система валідації (`estimate-validation.ts`)

Автоматично перевіряє кожен згенерований кошторис на 9 типів помилок:

#### A) Перевірка цін
- ❌ Нульові/від'ємні ціни (`unitPrice <= 0`)
- ⚠️ Підозріло низькі (`< 1 грн`)
- ⚠️ Підозріло високі (`> 100,000 грн`)
- ⚠️ "Круглі" ціни (`100, 500, 1000` - ознака галюцинації)

#### B) Перевірка кількостей
- ❌ Нульові/від'ємні кількості
- ⚠️ Нереалістичні кількості:
  - `500 розеток для 150 м²` - надмірно (норма: 0.3-0.5/м²)
  - `200 вимикачів для 100 м²` - надмірно (норма: 0.15-0.2/м²)

#### C) Перевірка wizard відповідності
- ⚠️ Розбіжність > 30% між wizard і кошторисом
- Приклад: wizard каже "45 розеток", AI згенерував 80 → WARNING

#### D) Перевірка заборонених позицій
- ❌ КРИТИЧНО: Демонтаж для shell building
- Якщо `demolitionRequired: false` → будь-яка позиція з "демонтаж" = БЛОК!

#### E) Перевірка розрахунків
- ❌ `totalCost ≠ quantity × unitPrice + laborCost`
- ❌ `sectionTotal ≠ sum(items.totalCost)`
- ❌ `summary.materialsCost ≠ sum(items materials)`

#### F) Перевірка дублікатів
- ⚠️ "Газоблок AEROC 300" і "Газоблок 300" - можливий дублікат

#### G) Перевірка пустих секцій
- ⚠️ Секція "Електрика" без позицій

#### H) Перевірка мінімальної кількості
- ⚠️ Занадто мало позицій для площі (< 0.8 позицій/м²)
- ⚠️ Занадто багато позицій (> 3 позиції/м²)

#### I) Перевірка наявності необхідних секцій
- ⚠️ Wizard каже "опалення газ", але секції опалення немає

**Використання**:

```typescript
import { validateEstimate } from '@/lib/estimate-validation';

const result = validateEstimate(estimateData, {
  area: 150,
  wizardData: wizardData,
  files: 27,
});

console.log(result.valid); // true/false
console.log(result.errors); // Критичні помилки
console.log(result.warnings); // Попередження
console.log(result.stats); // Статистика
```

**Інтеграція**: Валідація автоматично виконується в `/api/admin/estimates/generate` перед поверненням результату.


- Walls (gasblock, brick, insulation)

### 4. Підтримка специфікацій (`specification-parser.ts`)

Для проектів з детальними технічними специфікаціями (400-500 сторінок).

**Автоматична класифікація файлів**:
- **Плани** (для підрахунку): план електрики, план сантехніки, архітектурні креслення
- **Специфікації** (для деталей): технологічні карти, інструкції монтажу, вимоги стандартів

**Детекція специфікацій**:
- Назва містить: `специф`, `spec`, `технолог`, `інструкц`, `вимог`, `instruction`
- Розмір > 10 MB

**Витягує із специфікацій**:

```typescript
interface SpecificationData {
  materials: [
    { name: "Кабель ВВГ-нг 3×2.5 мм²", brand: "ПівденьКабель", quantity: "250 м" }
  ],
  methods: [
    {
      operation: "Монтаж розетки",
      tools: ["Перфоратор", "Рівень"],
      materials: ["Дюбель 6×40", "Клеми WAGO 2273-243"],
      steps: ["Розмітка", "Свердління", "Монтаж підрозетника"],
      standards: "ДСТУ-Н Б В.2.5-85:2013"
    }
  ],
  requirements: [
    {
      category: "Електрика",
      requirement: "Обов'язково використовувати негорючий кабель ВВГ-нг",
      critical: true,
      standard: "ДСТУ 7274:2012"
    }
  ]
}
```

**Використання в промпті**:

```typescript
const specContext = generateSpecificationContext(specificationData);
// Додається до промпту:
// "Використовуй цю специфікацію для деталізації:
//  - Точні марки: Кабель ВВГ-нг ПівденьКабель 3×2.5 мм²
//  - Методи: Монтаж з дюбелями 6×40, клеми WAGO
//  - Стандарти: ДСТУ 7274:2012"
```

**Результат**: AI генерує детальніші позиції з конкретними марками та методами монтажу.

## Як це працює разом

### 1. Користувач завантажує файли:
- `План електрики.pdf` (3 сторінки)
- `План сантехніки.pdf` (2 сторінки)
- `Специфікація матеріалів.pdf` (458 сторінок) ⬅️ НОВОЕ!
- `Технологічні карти монтажу.pdf` (127 сторінок) ⬅️ НОВОЕ!

### 2. Pre-analysis (`/api/admin/estimates/analyze`):
```
📂 Класифікація:
   - Плани: 2 файли (для підрахунку розеток, унітазів)
   - Специфікації: 2 файли (для деталей та інструкцій)

📊 Витягнуто зі специфікацій:
   - Матеріали: 327 позицій
   - Технологічні карти: 45 операцій
   - Вимоги: 89 (критичних: 23)
```

### 3. Generation (`/api/admin/estimates/generate`):
```
🤖 Gemini отримує:
   - Wizard дані (поверхи, розетки, матеріали стін)
   - Плани (візуальний аналіз PDF)
   - Текст зі специфікацій (марки, методи, вимоги)
   - Базу матеріалів (реальні ціни)
   - Temperature: 0.1 (детермінізм)
   - responseMimeType: "application/json" (strict format)

📝 Генерується кошторис → 180 позицій

🔍 Автоматична валідація:
   ✅ Ціни коректні (з бази матеріалів)
   ✅ Кількості реалістичні
   ✅ Немає заборонених позицій
   ✅ Розрахунки вірні
   ⚠️ 3 попередження (не критичні)

✅ Результат:
   - Детальні позиції з марками ("Кабель ВВГ-нг ПівденьКабель 3×2.5")
   - Правильні методи монтажу (згідно специфікації)
   - Реальні ціни (з бази даних)
   - Постійний результат (детермінізм)
```

## Підтримка

Якщо виникли питання або знайдено нові типи галюцинацій - додайте перевірку в `estimate-validation.ts`.

**Приклад додавання нової перевірки**:

```typescript
function validateNewCheck(estimate: any, errors: ValidationError[]) {
  estimate.sections?.forEach((section: any) => {
    section.items?.forEach((item: any) => {
      if (/* ваша умова */) {
        errors.push({
          type: 'ERROR',
          code: 'YOUR_ERROR_CODE',
          message: `Опис помилки`,
          section: section.title,
        });
      }
    });
  });
}
```
