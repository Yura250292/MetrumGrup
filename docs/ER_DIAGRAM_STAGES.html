Стиль MS Access. Лінії 1 — ∞ читаються: один рядок з лівої таблиці може мати багато повʼязаних рядків у правій. Жовтий бейдж PK — первинний ключ, синій FK — зовнішній ключ.

PK Первинний ключ

1 — ∞ Один-до-багатьох
kind жовте — дискримінатор PLAN/FACT
Серце системи — ProjectStageRecord Один рядок = один етап. Має ВОДНОЧАС поля plan* і fact*. Не дві таблиці, а одна з парами колонок.
FinanceEntry.kind — дискримінатор Один запис = одна проводка. Поле kind = PLAN | FACT розрізняє заплановане від фактичного. Привʼязка до етапу через stageRecordId.
Дельта (Δ) НЕ зберігається Різниця між Планом і Фактом обчислюється на льоту: fact − plan. Немає ризику розсинхрону між «джерелом» і «кешем».
Published-копії Поля publishedPlan*, publishedFact* — заморожений знімок для клієнта. Дозволяє редагувати draft не зачіпаючи опублікованого.
Firm — multi-tenant root Усе ізольовано по фірмі (Group vs Studio). Виняток — контрагенти можуть бути спільні (firmId = null).
Лінія Project → FinanceEntry FinanceEntry може належати проекту НАПРЯМУ (без етапу) — наприклад, загальні витрати. Тому і stageRecordId nullable.
