# Інструкція для AI: Намалювати бізнес-модель проекту Metrum Group

## Загальний опис проекту

**Metrum Group** - це система управління будівельними проектами з фокусом на автоматизацію кошторисування через AI (Gemini, OpenAI, Anthropic).

**Основна мета**: Автоматизувати створення та перевірку будівельних кошторисів, управляти проектами, клієнтами, платежами та звітністю.

---

## 1. ОСНОВНІ СУТНОСТІ (Entities)

### 👤 Користувачі (User)
- **Ролі:**
  - `SUPER_ADMIN` - повний доступ
  - `MANAGER` - управління проектами та кошторисами
  - `CLIENT` - перегляд своїх проектів
  - `ENGINEER` - технічна перевірка кошторисів
  - `FINANCIER` - фінансова перевірка
  - `USER` - базовий доступ

- **Поля:**
  - email, password, name, phone, avatar
  - Telegram інтеграція: telegramId, telegramUsername, telegramChatId
  - Статус: isActive

- **Зв'язки:**
  - Створює проекти (як менеджер)
  - Має проекти (як клієнт)
  - Створює кошториси
  - Перевіряє кошториси (інженер/фінансист)
  - Створює новини, фото-звіти, файли

---

### 🏗️ Проекти (Project)
- **Опис:** Будівельний проект (ремонт квартири, будинок, комерційна будівля)

- **Поля:**
  - title, slug, description
  - address (адреса об'єкту)
  - area (площа в м²)
  - startDate, endDate, actualEndDate
  - Фінанси: budget, totalCost, totalPaid, totalLabor, totalMaterials
  - Статус: `PLANNING`, `IN_PROGRESS`, `ON_HOLD`, `COMPLETED`, `CANCELLED`
  - Етап: `FOUNDATION`, `WALLS`, `ROOF`, `FINISHING`, `COMPLETED`

- **Зв'язки:**
  - Належить клієнту (User)
  - Управляється менеджером (User)
  - Має кошториси (Estimate[])
  - Має платежі (Payment[])
  - Має фото-звіти (PhotoReport[])
  - Має файли (ProjectFile[])
  - Має записи інвентарю (InventoryTransaction[])

---

### 📊 Кошториси (Estimate)
- **Опис:** Детальний розрахунок вартості робіт та матеріалів для проекту

- **Поля:**
  - number (унікальний номер), title, description
  - Версія (version) - для відслідковування змін
  - Фінанси:
    - totalMaterials (матеріали)
    - totalLabor (роботи)
    - totalOverhead (накладні витрати)
    - totalAmount (загальна сума)
    - discount (знижка)
    - finalAmount (фінальна сума)
    - finalClientPrice (ціна для клієнта з маржею)
  - Маржа:
    - profitAmount, profitMarginLabor, profitMarginMaterials, profitMarginOverall
  - Податки:
    - pdvAmount (ПДВ 20%)
    - esvAmount (ЄСВ 22%)
    - militaryTaxAmount (Військовий збір 1.5%)
    - profitTaxAmount (Податок на прибуток)
    - unifiedTaxAmount (Єдиний податок)
    - pdfoAmount (ПДФО 18%)
  - Статус: `DRAFT`, `SENT`, `APPROVED`, `REJECTED`, `REVISION`, `ENGINEER_REVIEW`, `FINANCE_REVIEW`
  - **Верифікація через AI (нова функція):**
    - verificationStatus: `passed`, `warnings`, `critical`, `not_verified`
    - verificationScore (0-100)
    - verificationResults (JSON з детальними результатами)
    - verifiedAt, verifiedBy (openai/anthropic)

- **Зв'язки:**
  - Належить проекту
  - Створений користувачем
  - Перевірений інженером (engineerReviewedBy)
  - Перевірений фінансистом (financeReviewedBy)
  - Має секції (EstimateSection[])
  - Має позиції (EstimateItem[])

---

### 📑 Секції кошторису (EstimateSection)
- **Опис:** Групування позицій кошторису за категоріями робіт

- **Категорії (17 стандартних):**
  1. Демонтажні роботи
  2. Земляні роботи
  3. Фундаментні роботи
  4. Стіни та перегородки
  5. Перекриття
  6. Дах
  7. Вікна та двері
  8. Електрика
  9. Сантехніка
  10. Стяжка підлоги
  11. Штукатурні роботи
  12. Шпаклювальні роботи
  13. Малярні роботи
  14. Плиткові роботи
  15. Підлогові покриття
  16. Оздоблення стель
  17. Інші роботи

- **Поля:**
  - title, sortOrder

- **Зв'язки:**
  - Належить кошторису
  - Має позиції (EstimateItem[])

---

### 📝 Позиції кошторису (EstimateItem)
- **Опис:** Окрема позиція в кошторисі (матеріал + робота)

- **Поля:**
  - description (опис роботи/матеріалу)
  - unit (одиниця: м², м³, шт, кг)
  - quantity (кількість)
  - unitPrice (ціна за одиницю)
  - laborRate (вартість роботи за годину)
  - laborHours (години роботи)
  - amount (загальна вартість = quantity × unitPrice + laborCost)
  - Маржа: customMarginPercent, marginAmount, priceWithMargin
  - Ручне редагування: isManualOverride

- **Зв'язки:**
  - Належить кошторису
  - Належить секції
  - Може бути пов'язана з матеріалом (Material)

---

### 🧱 Матеріали (Material)
- **Опис:** Довідник будівельних матеріалів

- **Поля:**
  - name, sku (артикул), category
  - unit (одиниця виміру)
  - basePrice (базова ціна)
  - laborRate (вартість робіт)
  - markup (націнка %)
  - description
  - isActive (активний/архівний)

- **Приклади категорій:**
  - Стінові матеріали (газоблок, цегла)
  - Підлогові матеріали (ламінат, плитка)
  - Лакофарбові матеріали (фарби, шпаклівки)
  - Бетон та розчини
  - Гіпсокартон та профілі

---

### 💰 Платежі (Payment)
- **Опис:** Платежі клієнтів за проекти

- **Поля:**
  - amount (сума)
  - date (дата платежу)
  - method: `CASH`, `CARD`, `TRANSFER`, `OTHER`
  - status: `PENDING`, `COMPLETED`, `FAILED`, `REFUNDED`
  - description

- **Зв'язки:**
  - Належить проекту
  - Пов'язаний з користувачем

---

### 📸 Фото-звіти (PhotoReport)
- **Опис:** Фотографії з об'єкту будівництва

- **Поля:**
  - title, description
  - date (дата зйомки)
  - stage (етап будівництва)
  - images (масив посилань на фото)
  - isPublic (публічний для клієнта)

- **Зв'язки:**
  - Належить проекту
  - Створений користувачем

---

### 📄 Файли проекту (ProjectFile)
- **Опис:** Документи, плани, акти

- **Типи файлів:**
  - `PHOTO_REPORT` - фото-звіти
  - `DOCUMENT` - документи
  - `PLAN` - креслення, плани
  - `COMPLETION_ACT` - акти виконаних робіт
  - `ESTIMATE` - кошториси

- **Поля:**
  - filename, path, size
  - mimeType
  - uploadedAt

---

### 📦 Інвентар (Inventory)
- **Опис:** Склад матеріалів

- **Поля:**
  - currentQuantity (поточна кількість)
  - minQuantity (мінімальна кількість для сповіщення)
  - location (локація)

---

### 🔄 Транзакції інвентарю (InventoryTransaction)
- **Опис:** Рух матеріалів (прихід/розхід)

- **Типи:**
  - `PURCHASE` - закупівля
  - `USAGE` - використання на проекті
  - `RETURN` - повернення
  - `ADJUSTMENT` - коригування

- **Поля:**
  - quantity (кількість)
  - unitCost (вартість за одиницю)
  - totalCost (загальна вартість)
  - notes (примітки)

---

### 📰 Новини (NewsArticle)
- **Опис:** Новини та оновлення для клієнтів

- **Поля:**
  - title, content
  - publishedAt
  - isPublished (опубліковано)

---

### 🔔 Сповіщення (Notification)
- **Опис:** Сповіщення для користувачів

- **Типи:**
  - Оновлення проекту
  - Нові платежі
  - Перевірка кошторису
  - Низький рівень інвентарю

- **Поля:**
  - title, message
  - isRead (прочитано)
  - createdAt

---

### 📊 Фінансові шаблони (FinancialTemplate)
- **Опис:** Шаблони для розрахунку податків та маржі

- **Поля:**
  - name, description
  - taxationType: `PDV`, `UNIFIED`, `NO_TAX`
  - globalMarginPercent (глобальна маржа)
  - logisticsCost (логістика)
  - categoryMargins (маржа по категоріях)

---

### 📝 Податкові записи (TaxRecord)
- **Опис:** Історія розрахунків податків

- **Поля:**
  - Всі податки (ПДВ, ЄСВ, ПДФО, військовий збір)
  - totalTaxAmount (загальна сума податків)
  - netProfit (чистий прибуток)
  - effectiveTaxRate (ефективна ставка)
  - calculationDetails (JSON з деталями)

---

### 🔍 Аудит логи (AuditLog)
- **Опис:** Журнал всіх дій користувачів

- **Поля:**
  - action (дія)
  - entityType (тип сутності)
  - entityId (ID сутності)
  - changes (JSON з змінами)
  - ipAddress, userAgent

## 2. БІЗНЕС-ПРОЦЕСИ

### 🎯 Процес 1: Створення кошторису через AI

```
1. Менеджер завантажує файли проекту (PDF, Excel, зображення планів)
   ↓
2. Вибирає тип проекту і категорії робіт
   ↓
3. AI (Gemini) аналізує файли:
   - Розпізнає текст з PDF/Excel
   - Аналізує зображення планів
   - Використовує Google Search для пошуку актуальних цін
   - Генерує кошторис (50-120+ позицій)
   ↓
4. OpenAI автоматично перевіряє згенерований кошторис:
   - Коректність розрахунків
   - Реалістичність цін (діапазон 2024-2025)
   - Повнота позицій
   - Логіка секцій
   - Специфікації матеріалів
   ↓
5. Показується оцінка 0-100 + список проблем
   ↓
6. Менеджер може:
   - Редагувати вручну
   - Використати AI редагування (refine)
   - Зберегти кошторис
   ↓
7. Інженер перевіряє технічну частину (ENGINEER_REVIEW)
   ↓
8. Фінансист перевіряє фінансову частину (FINANCE_REVIEW)
   ↓
9. Відправка клієнту (SENT)
   ↓
10. Клієнт затверджує (APPROVED) або відхиляє (REJECTED)
```

---

### 💼 Процес 2: Управління проектом

```
1. Створення проекту:
   - Менеджер створює проект для клієнта
   - Вказує адресу, площу, бюджет
   - Завантажує креслення, плани
   ↓
2. Кошторис:
   - Генерація через AI
   - Затвердження клієнтом
   - Стає бюджетом проекту
   ↓
3. Виконання робіт:
   - Етапи: FOUNDATION → WALLS → ROOF → FINISHING
   - Фото-звіти на кожному етапі
   - Використання матеріалів (InventoryTransaction)
   ↓
4. Платежі:
   - Клієнт платить по етапах
   - Відслідковування totalPaid vs budget
   ↓
5. Завершення:
   - Акти виконаних робіт
   - Фінальні фото
   - Статус COMPLETED
```

---

### 📦 Процес 3: Управління інвентарем

```
1. Закупівля матеріалів (PURCHASE):
   - Менеджер додає матеріали
   - Вказує кількість, вартість
   - Inventory збільшується
   ↓
2. Використання на проекті (USAGE):
   - Прив'язка до проекту
   - Inventory зменшується
   ↓
3. Сповіщення:
   - Якщо currentQuantity < minQuantity
   - Notification для менеджера
   ↓
4. Повернення невикористаних (RETURN):
   - Inventory збільшується
```

## 4. КЛЮЧОВІ ФІЧІ

### 🤖 AI Генерація кошторисів
- **Моделі:** Gemini (основна), OpenAI GPT-4o, Anthropic Claude Opus 4
- **Можливості:**
  - Розпізнавання PDF, Excel, Word, зображень
  - Google Search для актуальних цін (epicentrk.ua, prom.ua, budmagazin.ua)
  - Генерація 50-120+ позицій за 30-60 секунд
  - 6 шаблонів проектів: foundation, shell, turnkey, house_full, apartment_rough, custom

### ✅ AI Верифікація кошторисів (нова функція)
- **Модель:** OpenAI GPT-4o
- **Перевірки:**
  - Коректність математичних розрахунків
  - Реалістичність цін (діапазон ±30% від середньої)
  - Повнота позицій (чи не пропущені матеріали/роботи)
  - Логіка секцій (правильний порядок)
  - Специфікації матеріалів (марки, розміри)
- **Результат:** оцінка 0-100 + детальний список проблем

### 📱 Telegram інтеграція
- Бот для адміністраторів
- AI асистент для запитів до БД
- Сповіщення про події

### 📊 Звітність
- Фінансові звіти по проектах
- Розрахунок податків (ПДВ, ЄСВ, ПДФО, військовий збір)
- Маржинальність по категоріях
- Рентабельність проектів

### 📤 Експорт
- Кошториси в Excel, PDF

1. **ERD (Entity Relationship Diagram)**
   - Всі сутності та їх зв'язки
   - Кардинальність (1:1, 1:N, N:M)
   - Ключові поля

2. **Діаграма бізнес-процесів**
   - Флоу створення кошторису через AI
   - Флоу управління проектом
   - Флоу платежів

3. **Діаграма архітектури**
   - Frontend → Backend → Database
   - AI інтеграції (Gemini, OpenAI, Anthropic)
   - Зовнішні сервіси

4. **Матриця доступу**
   - Ролі × Функції

## ІНСТРУКЦІЯ ДЛЯ AI:

**Намалюй діаграму яка показує:**

1. **Всі основні сутності** (User, Project, Estimate, EstimateSection, EstimateItem, Material, Payment, PhotoReport, etc.)

2. **Зв'язки між ними** з кардинальністю:
   - User 1:N Project (клієнт має багато проектів)
   - User 1:N Project (менеджер управляє багатьма проектами)
   - Project 1:N Estimate
   - Estimate 1:N EstimateSection
   - EstimateSection 1:N EstimateItem
   - Material 1:N EstimateItem
   - Project 1:N Payment
   - Project 1:N PhotoReport
   - і т.д.

3. **Ключові атрибути** кожної сутності (не всі, а найважливіші)

4. **Бізнес-процес AI генерації кошторису** як окрему діаграму флоу

5. **Ролі користувачів** і їх взаємодію з системою

**Стиль діаграми:**
- Чисто, зрозуміло, професійно
- Кольори для групування: сині для користувачів, зелені для проектів, помаранчеві для фінансів, фіолетові для AI
- Стрілки з підписами типу зв'язку
- Можна використати Mermaid, PlantUML, або будь-який зручний формат

**Формат виводу:** Mermaid код або PlantUML код, який можна скопіювати та відрендерити.
