# My Tasks UI Repack Instructions

Дата: 2026-04-23

## Мета

Покращити UI розділу `Мої задачі`, не змінюючи існуючу бізнес-логіку.

Потрібно зробити подачу зрозумілішою:

- хто що має зробити
- що горить
- що чекає мого рішення
- що я делегував
- що блокує роботу
- які задачі варто виконувати зараз

## Важливе обмеження

Не ламати поточну логіку.

Залишити:

- поточні API
- `scope`
- `focus`
- `viewMode`
- drawer задачі
- створення задачі
- таймер
- завершення задачі
- фільтр по проєктах
- існуючі режими перегляду

Цей план про UI-перепакування, назви, ієрархію і подачу.

## Поточна проблема

Розділ уже має хорошу функціональну основу:

- режими перегляду
- секції
- таблиця
- список
- групування по проєктах
- групування по людях
- AI-помічник дня
- drawer задачі

Але UI ще виглядає трохи технічним.

Користувач бачить:

- `Мої`
- `Створені`
- `Стежу`
- `Всі`
- `Секції`
- `Таблиця`
- `Список`

А має бачити:

- що мені зробити зараз
- що прострочено
- що чекає мого рішення
- що я делегував
- де блокери

## Нова назва розділу

Замість:

- `Мої задачі`

Рекомендовано:

- `Моя робота`

або:

- `Робочий центр`

## Рекомендований підзаголовок

```text
Що потрібно зробити, що очікує рішення і що блокує команду
```

## Нова структура екрана

### 1. Header

Показати:

- назву розділу
- короткий опис
- кнопку `Нова задача`
- головну AI-дію

Рекомендована структура:

```text
Моя робота
Що потрібно зробити, що очікує рішення і що блокує команду

[Скласти план дня AI] [Нова задача]
```

## 2. Блок `Стан роботи`

Замість просто KPI-лічильників зробити блок, який пояснює стан роботи.

### Показувати

- `Активні`
- `Прострочені`
- `На сьогодні`
- `Очікують рішення`
- `Блокери`

### Поточні лічильники

Уже є:

- `assigned`
- `overdue`
- `dueToday`
- `completed`
- `unread`

### Що можна зробити без зміни API

Спочатку залишити ті самі дані, але краще назвати і подати:

- `assigned` → `Активні`
- `overdue` → `Прострочені`
- `dueToday` → `На сьогодні`

Для `Очікують рішення` і `Блокери` можна:

- або додати пізніше
- або рахувати приблизно на клієнті з уже завантажених задач

## 3. AI-помічник дня

Поточний AI-блок корисний, але його краще зробити менш шумним.

### Замість великого набору кнопок

Зробити одну головну кнопку:

```text
Скласти план дня
```

### Додаткові дії

Показувати як маленькі chips або dropdown:

- `Пріоритети`
- `Що зависло`
- `Що делегувати`
- `Закрити сьогодні`
- `Де я блокую команду`

### Візуальний принцип

AI має виглядати як робочий помічник, а не рекламний блок.

Тому:

- менше декоративного фону
- компактніша висота
- одна головна дія
- інші дії другорядні

## 4. Основний режим перегляду

Поточний `sections` режим залишити основним.

Але в UI не називати його `Секції`.

### Рекомендовані назви

Замість:

- `Секції`

Краще:

- `Робочі зони`

або:

- `По пріоритетах`

## 5. Робочі зони

Основна подача задач має бути через зрозумілі секції.

### Рекомендовані секції

- `Роби зараз`
- `Чекає мого рішення`
- `Делеговано`
- `Блокери`
- `На сьогодні`
- `Прострочене`

### Важливо

Не обов’язково одразу міняти логіку групування.

Можна почати з:

- перейменування секцій
- кращих підказок під назвами
- чіткішого порядку блоків

## 6. Toolbar розділити на дві зони

Зараз в одному toolbar змішані:

- режим перегляду
- scope
- завершені
- фільтри

Це варто розділити.

## Зона 1. `Перегляд`

Показує, як дивитись задачі.

```text
Перегляд:
[Робочі зони] [Таблиця] [Список] [По проєктах] [По людях]
```

## Зона 2. `Фільтри`

Показує, які задачі включені.

```text
Фільтри:
[Мені призначено] [Я поставив] [Я стежу] [Всі] [Завершені] [Проєкт]
```

## 7. Перейменування scope-кнопок

Поточні назви:

- `Мої`
- `Створені`
- `Стежу`
- `Всі`

Рекомендовані назви:

- `Мені призначено`
- `Я поставив`
- `Я стежу`
- `Всі задачі`

Це зрозуміліше для користувача.

## 8. Перейменування view modes

Поточні назви:

- `Секції`
- `Таблиця`
- `Список`
- `По проєктах`
- `По людях`

Рекомендовані назви:

- `Робочі зони`
- `Таблиця`
- `Список`
- `Проєкти`
- `Люди`

## 9. Рядок задачі

Рядок задачі має швидко відповідати:

- що зробити
- для якого проєкту
- хто відповідальний
- хто поставив
- який наступний крок
- чи є блокери
- коли дедлайн

## Дані, які вже можна використати

У `TaskItem` вже є:

- `title`
- `project`
- `status`
- `priority`
- `dueDate`
- `assignees`
- `createdBy`
- `firstUndoneChecklistItem`
- `incomingDepsCount`
- `outgoingDepsCount`
- `labels`
- `_count.subtasks`
- `_count.checklist`

## Що показувати в compact row

Основний рядок:

```text
[priority] Назва задачі                  [status] [deadline]
Проєкт · від кого · виконавці
Наступний крок: ...
```

Якщо є блокери:

```text
Блокує / Очікує залежність
```

## 10. Drawer задачі

Drawer не потрібно ламати.

Але UI можна покращити через порядок блоків.

### Рекомендований порядок

1. Назва задачі
2. Проєкт
3. Відповідальні
4. Статус / дедлайн / пріоритет
5. AI-помічник по задачі
6. Checklist
7. Залежності
8. Коментарі
9. Час / логи
10. Custom fields

### AI у drawer

Показати компактний блок:

```text
AI-помічник
[Поясни] [Розбий на кроки] [Наступний крок] [Повідомлення]
```

## 11. Empty state

Поточні empty states нормальні, але можна зробити більш робочими.

### Приклад

```text
Все під контролем
Немає активних задач у цій зоні.

[Створити задачу] [Показати всі]
```

## 12. Mobile подача

На мобільному не намагатись показати все.

### Mobile порядок

1. Заголовок
2. AI план дня
3. Стан роботи
4. Робочі зони
5. Фільтри в drawer / collapsible

### Toolbar mobile

Замість великої панелі:

- `Вигляд`
- `Фільтр`
- `Нова`

## 13. Що не робити

Не потрібно:

- прибирати існуючі view modes
- міняти API
- міняти модель задач
- переписувати drawer з нуля
- ховати AI повністю у чат
- робити AI блок занадто великим

## 14. Рекомендований порядок впровадження

### Крок 1

Перейменувати заголовок:

- `Мої задачі` → `Моя робота`

Додати підзаголовок.

### Крок 2

Перепакувати верх:

- `Стан роботи`
- `Скласти план дня AI`
- `Нова задача`

### Крок 3

Розділити toolbar:

- `Перегляд`
- `Фільтри`

### Крок 4

Перейменувати:

- `Секції` → `Робочі зони`
- `Мої` → `Мені призначено`
- `Створені` → `Я поставив`
- `Стежу` → `Я стежу`
- `Всі` → `Всі задачі`

### Крок 5

Покращити рядок задачі:

- додати `createdBy`
- додати `firstUndoneChecklistItem`
- додати dependency indicators
- краще показати виконавців

### Крок 6

У drawer додати компактний AI-блок по задачі.

## 15. Очікуваний результат

Після змін користувач має відкривати сторінку і швидко розуміти:

- що робити зараз
- які задачі горять
- які задачі чекають рішення
- які задачі він делегував
- де є блокери
- як AI може допомогти виконати задачу

## Підсумок

Поточна логіка вже достатньо хороша.

Головне покращення — не в переписуванні функціоналу, а в тому, щоб змінити подачу:

- менше технічних назв
- більше робочих формулювань
- чіткіше розділення перегляду і фільтрів
- AI як помічник у роботі, а не окремий декоративний блок

Ціль: зробити `Мої задачі` не списком задач, а зрозумілим робочим центром.
