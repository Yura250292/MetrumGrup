# Безпека проекту

## 🔒 Політика безпеки

### Захищені дані

Проект містить чутливу інформацію. **НІКОЛИ** не публікуйте в репозиторій:

- ❌ API ключі (Gemini, Claude, OpenAI)
- ❌ Паролі баз даних
- ❌ AUTH_SECRET ключі
- ❌ Персональні дані користувачів
- ❌ Фінансову інформацію

### ✅ Що вже захищено

- `.env` файл додано в `.gitignore`
- Паролі хешуються через bcrypt
- NextAuth.js для безпечної аутентифікації
- Role-based access control (RBAC)
- JWT токени для сесій

### 🛡️ Рекомендації для продакшну

#### 1. Змінні оточення
```bash
# Згенеруйте надійний AUTH_SECRET
openssl rand -base64 32

# Використовуйте сильні паролі для БД
# Не використовуйте postgres:postgres!
```

#### 2. База даних
- Використовуйте SSL з'єднання для PostgreSQL
- Регулярно робіть бекапи
- Обмежте доступ до БД тільки з довірених IP

#### 3. API ключі
- Зберігайте в змінних оточення хостингу
- Ніколи не коммітьте в git
- Регулярно ротуйте ключі
- Використовуйте обмеження rate-limiting

#### 4. Аутентифікація
- Змініть всі тестові паролі
- Встановіть мінімальну довжину пароля (8+ символів)
- Використовуйте 2FA для адміністраторів
- Налаштуйте auto-logout через неактивність

#### 5. HTTPS
- Використовуйте тільки HTTPS на продакшні
- Налаштуйте HSTS заголовки
- Використовуйте SSL сертифікати

## 🚨 Повідомлення про вразливості

Якщо ви знайшли вразливість безпеки:

1. **НЕ** створюйте публічний issue
2. Напишіть на [security@metrum.group](mailto:security@metrum.group)
3. Опишіть проблему детально
4. Дочекайтесь відповіді перед публічним розголошенням

## 📋 Checklist перед деплоєм

- [ ] Змінено AUTH_SECRET
- [ ] Змінено паролі БД
- [ ] API ключі в змінних оточення
- [ ] Видалено всі console.log з чутливими даними
- [ ] Налаштовано HTTPS
- [ ] Змінено тестові паролі користувачів
- [ ] Налаштовано бекапи БД
- [ ] Обмежено доступ до БД
- [ ] Налаштовано rate limiting
- [ ] Перевірено CORS налаштування
- [ ] Налаштовано CSP заголовки
- [ ] Увімкнено логування помилок
- [ ] Налаштовано моніторинг
